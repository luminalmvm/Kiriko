audit: T16/T17/T24/T21/T4/A1/A3/T25/T14/A4, EC1-4, TL3/TL4/T22/T23/T11, the T18 and T19

## Verified clean

1. **No-panic rule** — no `unwrap`/`expect`/`panic!` outside `#[cfg(test)]` in
   `lumit-core`, `lumit-gpu`, `lumit-eval` (all sweep hits sit inside test modules).
2. **Theme discipline** — no hex/`Color32` literals outside `theme.rs`/`icons.rs`.
3. **Cache-key honesty** — `lumit-eval` hashes effect parameters generically by evaluated
   value (`EffectValue::*` walk), so every parameter this sprint added (Vignette `ramp`,
   RGB-split/spectral `tints`, Shake `motion_blur`/`mb_amount`, Datamosh
   `displacement`/`bloom`/`reset_interval`, Echo's renumbered `mode`) feeds the frame key
   with no per-effect wiring. `blend_tag` gained stable bytes 10–25 for the new blend
   modes (never reused).
4. **K-031 preview == export** — the two duplicate `BlendMode → gpu::Blend` mappers were
   collapsed to one (`shell::inspector::blend_of`; export delegates). The compositor's
   snapshot blends — previously oracle-less — now have a GPU test against a Rust reference
   (`perceptual_blend_modes_match_the_reference_formula`). The T18/T19 kernels ship with
   CPU/GPU oracles (both measured worst 1 fp16 ULP, bound ≤ 2).
5. **Snapshot blend routing** — `fs_layer_snapshot` boundaries re-checked: modes 4/5/6
   (lighten/darken/subtract) run linear; 0–3 and 7–22 run encoded via `blend_encoded`;
   no mode falls through to the wrong branch.
6. **Serde/compat** — `BlendMode` serialises by variant name (old projects load; new
   variants are additive). Every new field on UI state is session-only (not serialised).
   The echo-mode renumber and datamosh param reshape rely on the pre-release no-migration
   policy, and both are recorded (docs/02, tracker).
7. **Tests** — full workspace green at HEAD: 571 non-GPU + 55 GPU (single-threaded) = 626.
   fmt and `clippy --workspace --all-targets -D warnings` clean.
8. **`DefaultHasher` regression check** — one use (`audio_jobs_signature`) is
   session-only change detection, never persisted: safe. A guard comment was added so it
   is not copied into persisted paths (the prior audit's `cache_root_for` bug class).

# Pass-5 audit — 2026-07-20 (after the 26-item overnight batch + TL2)

Scope: the pass-5 code (K-166/K-167/K-168, the row/column rework, the new controls) plus a
fresh cache-honesty and wiring sweep of the application. Gates at time of writing: fmt,
clippy -D warnings, 628+ workspace tests, GPU oracle suite, no-hex lint — all green.

## Fixed this audit

- **Solo was missing from the frame cache key (real bug, K-105 × Kura).** `feed_comp` gated
  layers on visibility and span only, while both renderers also gate on solo — so soloing a
  layer changed the pixels but not the key, and the RAM/disk frame cache would replay the
  pre-solo picture on any frame it already held. Fixed by gating exactly as the renderers do
  (`any_solo && !layer.switches.solo` skips), so solo states with identical contributions
  (e.g. every contributing layer soloed) still share keys. Regression test:
  `solo_gates_layers_out_of_the_key` (fails without the gate).
- **The hidden audio waveform had no way back on** from the menus (T25 asked for one): the
  Window menu now carries the toggle beside the lane right-click one.
- **docs/07-UI-SPEC §4.2** described the pre-TL2 column layout; now records the shipped
  five-cluster arrangement and what still awaits machinery.
- **Lock/label round-trip test** added to the store suite (commit → undo → undo restores
  both, one step each).

## Checked and found sound

- **Quality tiers and the frame key.** Suspected the same class of bug as solo for the
  preview quality dial (Half/Quarter/Auto decodes cached under a full-quality key) — it is
  not one: `PreviewStamper` folds the decode target width into every footage stamp
  (`path#w{target}`, draft pinned to false because draft frames are never banked), so each
  quality tier keys separately by design (docs/06 §5.2). A speculative "decode fills at full
  res" change was written and then reverted as wrong: it would have stored full-res pixels
  under current-tier keys, making cached frames sharper than live-rendered neighbours.
- `Layer.label` and `switches.locked` stay out of the frame key (feed_layer hashes fields
  individually) — organisational state never retires cached frames.
- `label` is the only new serialized field and carries `#[serde(default)]`; `locked`
  predates this pass. Old projects load; a stored Posterize Scope value is ignored (K-166).
- All five user-facing apply sites route through `fx::instantiate_for_raster`; remaining
  plain `instantiate` calls are test/preset fixtures.
- Posterize kind checks agree at all three orchestration sites (temporal.rs, draws.rs,
  export.rs: `LayerKind::Adjustment`).
- No panics in the new engine code (maths.rs, temporal.rs); hex lint clean; no new locks
  or async.

## Honest improvement notes (for the owner's next pass; no action taken)

- **Narrow outlines pile the TL2 switch clusters up** — below roughly 420 px the
  right-anchored groups clamp against the names. AE solves this by dropping whole columns
  as width shrinks; worth doing once column visibility becomes configurable (07 §4.2 keeps
  that open).
- **Lock guards timing/order only** (bar, trim, reorder). Keyframes, values, effects and
  delete still work on a locked layer. AE locks everything. Deliberate v1 choice (documented
  in K-168) — flag if full lock is wanted.
- **Label palette borrows theme roles.** Eight colours from the existing theme struct keep
  the no-hex rule, but they shift with the colour scheme; a dedicated user-editable label
  palette belongs with the planned theme-customisation work.
- **`cache_bar` skips comps over 2 400 frames** (documented S-budget debt) — a 40-minute
  comp shows no cache bar. The incremental evaluator bar remains the real fix.
- **panel.rs is 2 000 lines again** after TL2; export.rs ~1 960. Both are candidates for
  the next file-hygiene split (K-007 readability).
- **The parent dropdown cycles silently** — a parent choice that would create a cycle is
  simply absent from the list; fine, but a greyed entry with a tooltip would teach the rule.
- **Waveform/grid toggles are app-level, not per-project**, unlike the outline width
  (per-project egui persistence). Harmless asymmetry; unify if per-project workspace state
  grows.
